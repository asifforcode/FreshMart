# FreshMart - Retail Insights

This project provides a data-driven solution for FreshMart, a regional supermarket chain, to optimize its inventory management.

## Business Case

FreshMart is facing financial losses due to inefficient inventory control. They often overstock items that have low demand and run out of popular items. This project aims to provide a "Stock Health Report" to help the inventory manager make better decisions.

## Problem Statement

The primary goal is to create a report that identifies two key issues:
1.  **Expiring Soon:** Products that are nearing their expiration date.
2.  **Dead Stock:** Products that have not been sold in the last 60 days.

## Database Schema

The database consists of three main tables:
*   `Products`: Contains details about each product, including stock count and expiry date.
*   `Categories`: Groups products into different categories.
*   `SalesTransactions`: Records all sales, linking products to transactions.

## SQL Queries

The `schema.sql` script includes the following reports:

1.  **Expiring Soon Report:** Lists products that will expire within the next 7 days and have a stock count greater than 50.
2.  **Dead Stock Analysis:** Identifies products that have not been sold in the last two months.
3.  **Revenue Contribution:** Calculates the total revenue generated by each product category in the last month, and identifies the top-performing category.

## How to Use

1.  Execute the `schema.sql` script in your SQL database to create the tables and populate them with sample data.
2.  Run the queries provided in the script to generate the reports.
